# 210 정형외과 병동관리시스템 - 요구사항 명세서

**최종 수정일:** 2026년 4월 8일  
**시스템 버전:** v1.0 (PWA)  
**플랫폼:** Web (PWA), Android, iOS, Windows/Mac Desktop

---

## 1. 프로젝트 개요

### 1.1 시스템 목표
210 정형외과 병동에서 병상 현황, 환자 정보, 퇴원 일정을 실시간으로 관리하고 중요한 이벤트(퇴원 D-1, 장기입원, 시간 초과)를 자동으로 감지하여 의료진에게 알림을 제공하는 통합 관리 시스템

### 1.2 핵심 가치
- **실시간 동기화**: 30초 간격 자동 동기화로 여러 기기 간 데이터 일관성 보장
- **자동 알림**: 퇴원 예정, 장기입원, 시간 초과 조건을 자동 감지
- **오프라인 지원**: 인터넷 끊김 시에도 로컬 데이터 보존 및 자동 동기화
- **PWA 기술**: 앱 설치 없이 홈화면 추가로 네이티브 앱처럼 사용
- **무료 운영**: Firebase Spark(무료) 요금제로 운영

## 2. 기능 요구사항 (FRD)

### 2.1 사용자 인증 (Authentication)

#### 2.1.1 회원가입 (Sign Up)
- **요구사항**
  - 이메일과 비밀번호로 새 계정 생성
  - 비밀번호 확인 입력 필수
  - Firebase Auth를 통한 안전한 저장
  - 회원가입 실패 시 오류 메시지 표시

#### 2.1.2 로그인 (Sign In)
- **요구사항**
  - 이메일과 비밀번호 입력
  - 자동 로그인: 최초 로그인 후 로컬 스토리지에 세션 유지
  - 브라우저 종료 후에도 로그인 상태 유지
  - 로그인 실패 시 상세 오류 메시지

#### 2.1.3 로그아웃 (Sign Out)
- **요구사항**
  - 로그아웃 전 확인 창 표시
  - 세션 종료 및 로컬 스토리지 클리어
  - 로그아웃 후 로그인 페이지로 리다이렉트

---

### 2.2 병상 관리 (Bed Management)

#### 2.2.1 병상 상태 관리
- **상태 목록 및 정의**
  | 상태 | 설명 | 색상 | 주용도 |
  |------|------|------|--------|
  | `empty` | 비어 있음 | 회색 | 빈 병상 |
  | `admitted` | 입원 중 | 파란색 | 현재 입원 환자 |
  | `morning_discharge` | 오전 퇴원 예정 | 주황색 | 12시까지 퇴원 예정 |
  | `afternoon_discharge` | 오후 퇴원 예정 | 주황빨강 | 18시까지 퇴원 예정 |
  | `afternoon_admission` | 오후 입원 예정 | 초록색 | 오후 입원 예정 |
  | `next_day_discharge` | 다음날 퇴원 예정 | 보라색 | 내일 퇴원 예정 |
  | `provisional_discharge` | 가퇴원 | 분홍빨강 | 임시 퇴원 상태 |

- **요구사항**
  - 각 병상은 ID, 병상번호, 상태, 환자정보를 포함
  - 병상 클릭 시 상세 편집 모달 열기
  - 상태 변경 시 즉시 저장 및 동기화
  - 병상 숫자(예: 501호, 502호)는 설정으로 커스터마이징 가능

#### 2.2.2 병상 카드 (Bed Card)
- **요구사항**
  - 병상번호, 상태 배지, 환자 정보 표시
  - 환자명, 나이, 성별, 진단명 표시
  - 입원일, 퇴원일, D-day 표시
  - 메모(특이사항) 표시
  - D-1 배지: 내일 퇴원 환자 표시
  - 장기 배지: 7일 이상 입원 환자 표시
  - 시간 초과 시 맥박 애니메이션 표시
  - 퀵 퇴원 버튼 제공

#### 2.2.3 병상 편집 모달
- **요구사항**
  - 병상 상태 선택 (7가지 상태 모두 표시)
  - 상태가 empty가 아닐 때만 환자 정보 입력 활성화
  - 환자 정보 입력 필드:
    - 환자명 (필수)
    - 나이 (선택)
    - 성별 (남/여 선택)
    - 입원일 (Date picker)
    - 퇴원일 (Date picker, 자동으로 D-day 계산)
    - 진단명 (텍스트)
    - 메모 (자유 텍스트)
    - 수술 환자 체크박스
  - 퇴원 상태(morning/afternoon/next_day_discharge)일 때 퇴원 체크리스트 제공:
    - 처방전 발행
    - 보호자 연락
    - 약 수령
    - 퇴원 교육
    - 외래 예약
    - 병실 정리
  - 저장 및 취소 버튼
  - "병상 비우기" 버튼 (상태를 empty로 변경)

---

### 2.3 환자 정보 관리 (Patient Information)

#### 2.3.1 기본 환자 정보
- **저장 필드**
  - 이름, 나이, 성별
  - 입원일, 퇴원 예정일
  - 진단명, 특이사항 메모
  - 수술 여부 플래그

#### 2.3.2 입원 기간 계산
- **요구사항**
  - 입원일 기준으로 현재까지의 입원 일수 자동 계산
  - 퇴원 예정일 기준으로 D-day 자동 계산 (D-1, D-2, ... D+1, D+2, ...)
  - 매일 자정(00:00)에 기준일자 변경

### 2.4 알림 시스템 (Alert System)

#### 2.4.1 퇴원 D-1 알림
- **조건**: 퇴원 예정일이 내일인 환자
- **표시**: 빨간색 배너, "D-1" 배지
- **음성**: 띵동 톤 (880Hz → 1320Hz)
- **액션**: "확인(닫기)" 버튼으로 그날은 다시 표시 안 함

#### 2.4.2 장기입원 알림 (7일 초과)
- **조건**: 입원일로부터 7일 이상 경과한 환자
- **표시**: 노란색 배너, "장기" 배지
- **음성**: 띵띵 톤 (660Hz 2회)
- **액션**: "확인(닫기)" 버튼으로 그날은 다시 표시 안 함

#### 2.4.3 오전 퇴원 시간 초과 알림
- **조건**: 상태가 morning_discharge인데 현재 시각이 설정된 오전 마감 시간(기본 12:00) 초과
- **표시**: 파란색 배너, 병상 카드 맥박 애니메이션, "시간 초과" 표시
- **음성**: 삐빅 톤 (440Hz + 550Hz)
- **액션**: "퇴원 완료" 버튼으로 빠른 퇴원 처리

#### 2.4.4 알림 음성
- **요구사항**
  - Web Audio API를 사용한 합성음 생성 (외부 파일 불필요)
  - 각 알림 유형별 고유 톤 제공
  - 음소거 기능은 현재 미제공 (향후 기능)

### 2.5 모달 기능

#### 2.5.1 공지사항 모달 (Notice Modal)
- **요구사항**
  - 공지 제목, 내용, 긴급 여부 입력
  - 공지 등록 시 현재 시각 자동 기록
  - 공지 작성자를 "관리자"로 표시
  - 긴급 공지는 빨간색 배너로 강조
  - 시간순 역순 정렬 (최신순)

#### 2.5.2 수술 일정 모달 (Surgery Modal)
- **요구사항**
  - 병상, 수술 시간, 수술명, 집도의 입력
  - 시간순 정렬
  - 완료/취소 토글 버튼
  - 완료된 수술은 회색으로 표시

#### 2.5.3 통계 모달 (Stats Modal)
- **표시 항목**
  - 전체 병상 수
  - 점유율 (%)
  - 현재 입원 환자 수
  - 평균 재원 기간 (일)
  - 상태별 인원수 (표와 프로그래스바)

#### 2.5.4 병동 설정 모달 (Settings Modal)
- **커스터마이징 항목**
  - 병동 이름 (기본: "210정형외과병동관리시스템")
  - 총 병상 수 (기본: 20)
  - 병상 시작 번호 (기본: 1)
  - 병상 prefix (기본: 빈 문자열, 예: "5" 입력 시 501호, 502호, ...)
  - 오전 마감 시간 (기본: 12:00)
  - 오후 마감 시간 (기본: 18:00)

### 2.6 필터링 및 정렬

#### 2.6.1 상태별 필터
- **요구사항**
  - "전체" 탭: 모든 병상 표시
  - 상태별 탭: 각 상태에 해당하는 병상만 표시
  - 필터 변경 시 즉시 화면 업데이트
  - 각 탭에 해당 병상 수 표시

#### 2.6.2 정렬
- **요구사항**
  - 기본 정렬: 병상 번호 오름차순
  - 필터링된 상태에서도 병상 번호순 유지

---

### 2.7 데이터 동기화 (Data Synchronization)

#### 2.7.1 자동 동기화
- **요구사항**
  - 30초 간격 폴링으로 서버에서 최신 데이터 수신
  - 로컬 저장소에 자동 저장 (localStorage)
  - Firebase Firestore에 백업 저장
  - 탭 활성화 시 즉시 동기화
  - 인터넷 연결 복구 시 즉시 동기화

#### 2.7.2 변경사항 저장
- **요구사항**
  - 사용자가 수정 후 1.5초 내에 자동 저장 (디바운싱)
  - 저장 상태 표시 (저장 중 → 저장됨 → idle)
  - 저장 실패 시 로컬에는 저장되어 있음 (데이터 손실 없음)

#### 2.7.3 동기화 상태 표시
- **상태 아이콘 (화면 상단 센터)**
  - `saving` (노란색): 저장 중... (회전 아이콘)
  - `saved` (초록색): 저장됨 (체크 아이콘, 2초 후 사라짐)
  - `error` (빨간색): 저장 실패 (X 아이콘, 클릭해서 재시도 불가능)
  - `offline` (회색): 오프라인 (⊘ 아이콘)

## 3. 비기능 요구사항 (NFR)

### 3.1 성능
- **로딩 시간**: 3초 이내
- **응답 속도**: 사용자 입력 후 100ms 이내 화면 업데이트
- **메모리 사용**: 병상 20개 기준 10MB 이하
- **배터리 소비**: 30초 폴링으로 최소화

### 3.2 보안
- **인증**: Firebase Auth (암호화된 비밀번호)
- **접근 제어**: 로그인한 사용자의 데이터만 접근 가능
- **데이터 암호화**: HTTPS 통신 필수
- **로컬 스토리지**: 민감한 정보는 저장하지 않음 (비밀번호 제외)

### 3.3 사용성
- **반응형 디자인**: PC, 태블릿, 모바일 모두 최적화
- **접근성**: 시각 장애인을 위한 색상 대비 (WCAG AA 준수)
- **다국어**: 현재 한국어 전용 (향후 다국어 지원 가능)

### 3.4 유지보수성
- **코드 구조**: React 컴포넌트 기반
- **상태 관리**: useState 및 useRef로 간단하게 관리
- **Firebase**: Spark(무료) 요금제로 저비용 운영

### 3.5 확장성
- **병상 수 변경**: 설정에서 병상 수 변경 가능 (재배포 불필요)
- **알림 추가**: 새로운 조건을 쉽게 추가 가능
- **모달 추가**: 새로운 모달을 쉽게 확장 가능

## 4. 제약사항 (Constraints)

### 4.1 기술 제약
- **브라우저**: Chrome, Safari, Edge 최신 버전 필수
- **React 버전**: 18.x (CDN 사용)
- **Firebase**: Spark(무료) 요금제

## 8. 용어 정의 (Glossary)

| 용어 | 정의 |
|------|------|
| PWA | Progressive Web App (홈화면 추가 가능한 웹앱) |
| Firebase | Google이 제공하는 백엔드 서비스 (인증, DB, 호스팅) |
| Spark | Firebase 무료 요금제 |
| Firestore | Firebase의 NoSQL 클라우드 데이터베이스 |
| 동기화 | 로컬과 서버 간 데이터 일관성 유지 |
| 폴링 | 주기적으로 서버에서 데이터 요청 |
| 디바운싱 | 연속된 사용자 입력을 일정 시간 뒤에 한 번만 처리 |
| D-day | 목표일(퇴원일)로부터의 경과일 |
| 장기입원 | 7일 이상 입원 상태 |
